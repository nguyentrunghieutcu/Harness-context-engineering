Tool Playbook

Descriptions, examples, and common use cases for the 13 MCP tools.
retrieve_context
Builds compressed RAG context from files. Use before debugging, refactoring, or explaining a module.
compact_conversation
Compacts long chat history into a summary block. Use before continuing long-running work.
reindex_paths
Refreshes the chunk index for changed paths. Use after large edits or newly created files.
invalidate_cache
Clears cached retrieval results. Use when output feels stale after branch or file changes.
memory_save
Stores decisions, project rules, and workflows in episodic, semantic, or procedural memory.
memory_search
Finds saved memory by meaning. Use to recover prior decisions before changing code.
memory_inject
Adds relevant memory to a prompt without file retrieval. Use for convention-heavy tasks.
memory_delete
Removes stale or wrong memory by key and type.
memory_list
Lists recent memory entries. Use for audits before cleanup.
memory_evict
Prunes least-used memory while keeping the most valuable entries.
memory_stats
Shows counts, tokens, and hits per memory tier.
estimate_tokens
Measures prompt size. Use before injecting long text into a model.
get_token_budget
Shows context window, reserves, and inject budget for a model.
retrieve_context({ "base_prompt": "Use retrieved context.", "paths": ["src/", "server.py"], "query": "optimize auth flow", "model": "gpt-5.5", "output_mode": "structured" })
compact_conversation({ "messages_json": "[{\"role\":\"user\",\"content\":\"...\"}]", "retain_turns": 2, "model": "gpt-5.5" })
memory_save({ "key": "auth-refresh-token-policy", "value": "Rotate refresh tokens after use.", "mtype": "semantic", "tags": "auth,security" })
get_token_budget({ "model": "gpt-5.5" }) reindex_paths({ "paths": ["src/auth/"] })
